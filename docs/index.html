The Problem

Your monitoring tells you when services are down. But who monitors the monitoring?

  • Did that nightly backup actually run?
  • Is your alerting pipeline actually delivering?
  • When a cron job silently fails, how long until you notice?

The Solution

Schedule Verification

Define expectations: "this job runs every 24h with 30min tolerance." Rewire detects missed runs, long-running jobs, and overlapping executions.

Alert Path Testing

Rewire sends synthetic test alerts and requires acknowledgment. If your alert path breaks, you'll know before a real incident.

Formal Specification

Built with Quint/TLA+ formal methods. Runtime invariant checker ensures the system behaves as specified.

Evidence-Based

Every violation includes evidence. No false positives from guesswork—only facts with timestamps.

Epistemic Contract

Rewire claims only what it can prove:

  • An expectation exists with explicit parameters
  • Observations arrived (with timestamps)
  • Reality matches or violates declared constraints

Rewire refuses to claim:

  • That a job's output is correct
  • That a human noticed or acted
  • That "delivery" implies "awareness"

Quick Start

# Install
pip install rewire-verify

# Start server
rewire --db rewire.db --init-db --base-url http://localhost:8080

# Create expectation
rewire-ctl --base-url http://localhost:8080 --admin-token YOUR_TOKEN \
  new-schedule --name "nightly-backup" --email you@example.com \
  --expected-interval-s 86400 --tolerance-s 1800

# Instrument your job
curl -X POST 'http://localhost:8080/observe/EXP_ID' -d kind=start
# ... do work ...
curl -X POST 'http://localhost:8080/observe/EXP_ID' -d kind=end
